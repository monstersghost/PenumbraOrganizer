# Penumbra Organizer v0.3.4.1-beta

## Changes since v0.3.4-beta

- **Fixed: mod classification broke on Penumbra 1.7.0+.** Penumbra 1.7.0 merged
  `default_mod.json` and every `group_*.json` into a single `meta.json` per mod (an automatic
  migration Penumbra runs on update). Because the app was still looking for those separate files,
  every mod on an updated Penumbra install would silently show up as "Others" instead of its real
  category (Gear, Weapon, Hair, etc.) — no error, just wrong classification. The app now reads the
  new combined `meta.json` layout directly, so classification works correctly again on Penumbra
  1.7.0+ (and still supports older installs that haven't updated yet).
- **New: full folder names in the Current Mods folder tree.** The "Current folder tree" panel
  used to clip long folder names with no way to see the rest. Long names now wrap onto multiple
  lines, hovering any row shows the full path as a tooltip, and the panel itself can be
  drag-resized wider via a splitter between it and the mods list.

## Known issues

- Live *creation* of new orphaned `mod_filesystem/organization.json` folders is still not fixed,
  carried over from earlier betas. Folder Cleanup / Advanced Cleanup can prune orphaned folders
  that already exist, but the original trigger that creates new ones hasn't been reproduced on any
  test install yet. See
  [docs/KNOWN_ISSUE_EMPTY_FOLDERS_AFTER_RESORT.md](../KNOWN_ISSUE_EMPTY_FOLDERS_AFTER_RESORT.md).
- A report of certain special characters in folder names causing "invalid changes" during Review
  Changes is still under investigation. It hasn't been reproduced yet with plain ASCII `-`/`--`
  across any part of the app tested so far. If you hit this, please share the exact folder or
  creator name involved (copied, not retyped) so it can be tracked down.
