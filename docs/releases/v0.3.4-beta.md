# Penumbra Organizer v0.3.4-beta

## Changes since v0.3.3-beta

- **New: one-click self-update.** "Check for Updates" now has a companion "Update Now" button.
  When a newer release is available, clicking it downloads the release, verifies it against the
  published SHA-256 checksums, and applies it automatically: the app closes, swaps its own files,
  and reopens as the new version. Your current manual-organizing session is saved automatically
  before the restart, so in-progress work isn't lost. If anything fails at any step (download,
  verification, or applying the update), nothing is touched and the app keeps running normally;
  a clear error is shown instead.
  - **Note for anyone updating from v0.3.3-beta or earlier:** those installs don't have the
    small helper program this feature depends on yet, so clicking "Update Now" from v0.3.3-beta
    will still just open the release page in your browser, same as before. Starting from this
    version (v0.3.4-beta) onward, "Update Now" will fully apply future updates automatically.
- **New: Heliosphere-managed mods are now automatically protected.** If Penumbra Organizer detects
  mods managed by the Heliosphere client, it protects them by default (so they're never moved,
  reclassified, or touched by Apply) and clearly marks them with a "Heliosphere" badge and a
  one-time reminder banner after scanning, so it's never a silent change. The banner includes a
  "Protect/Unprotect All" button to flip every Heliosphere-managed mod's protection at once, in
  case you want to organize them manually instead. (Closes
  [#12](https://github.com/monstersghost/PenumbraOrganizer/issues/12).)
- **Fixed: the workbook export's "protected" column could be wrong.** Exporting a workbook after
  protecting mods or folders in the app now correctly reflects which mods are actually protected.
  Previously the exported column could show every mod as unprotected regardless of what you'd
  actually protected in the app.
- Clarified Linux/Wine support in the project docs: the only confirmed working launch path is
  XIV Launcher → Wine → Open Wine Explorer (Run apps in the same prefix).

## Repackaged 2026-07-19

Windows Defender was flagging the original download as `Trojan:Win32/Tecabans.STV!cl`, a false
positive. The cause was the single-file, self-extracting exe format the app was built with, which
antivirus heuristics tend to treat as suspicious regardless of what's actually inside it.

The download has been rebuilt as a plain multi-file package instead. Same v0.3.4-beta code, same
features, same known issues below, just packaged differently. If you already downloaded and ran
the app before this date without Defender complaining, there's nothing to do. If Defender blocked
your download, re-downloading the zip below should resolve it.

The checksums section further down reflects the new package.

## Known issues

- Live *creation* of new orphaned `mod_filesystem/organization.json` folders is still not fixed,
  carried over from v0.3.3-beta. Folder Cleanup / Advanced Cleanup can prune orphaned folders that
  already exist, but the original trigger that creates new ones hasn't been reproduced on any test
  install yet. See
  [docs/KNOWN_ISSUE_EMPTY_FOLDERS_AFTER_RESORT.md](../KNOWN_ISSUE_EMPTY_FOLDERS_AFTER_RESORT.md).
- A report of certain special characters in folder names causing "invalid changes" during Review
  Changes is still under investigation. It hasn't been reproduced yet with plain ASCII `-`/`--`
  across any part of the app tested so far. If you hit this, please share the exact folder or
  creator name involved (copied, not retyped) so it can be tracked down.
